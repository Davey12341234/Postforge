PostForge — where to put Railway secrets for "npm run deploy:check"
================================================================

If Cursor/VS Code will not show or open deploy\secrets.preview.env, it is usually
because gitignored files are hidden. Workarounds:

1) HOME FILE (recommended — always visible in Explorer)
   Windows:  %USERPROFILE%\.postforge-deploy-check.env
   Example:  C:\Users\YourName\.postforge-deploy-check.env
   Copy deploy\secrets.preview.env.example, fill values, save under that name.
   Then run:  npm run deploy:check

2) ABSOLUTE PATH VIA ENVIRONMENT VARIABLE
   PowerShell:
     $env:DEPLOY_CHECK_FILE = "C:\secure\railway.env"
     npm run deploy:check
   (Also accepted: POSTFORGE_DEPLOY_CHECK_FILE, POSTFORGE_SECRETS_FILE)

3) REPO FILES (gitignored — IDE may hide them)
   deploy\secrets.preview.env
   .env.production.local

4) EXPLICIT FLAG
   node scripts\check-deploy-readiness.mjs --file C:\path\railway.env

This repo includes .vscode\settings.json so gitignored files can show in the
sidebar while node_modules stays collapsed via files.exclude.
